## 1. 專案概述

### 1.1 背景與動機

學生在學習過程中，往往使用分散的工具（紙本、Word、不同 App）記錄各科筆記，導致內容難以整理、搜尋與分享。讀書筆記本系統旨在提供一個集中化、結構化的線上筆記平台，讓學生可依科目分類管理筆記，並輕鬆統整重點與分享內容。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **學生** | 主要使用者，需記錄、整理各科目的讀書筆記 |
| **管理者** | 負責維護系統、管理使用者與內容 |

### 1.3 核心價值主張

- 📂 依科目分類，筆記脈絡一目瞭然
- ✏️ 透過網址即可協作編輯，無須帳號登入
- 📊 自動統整摘要，幫助學生快速複習
- 🔍 全文搜尋，快速找到關鍵內容

## 2. 功能需求

### 2.1 主要功能與使用者故事

#### 功能 1：科目筆記輸入

> 作為**學生**，我希望能**在不同科目下新增與編輯筆記內容**，以便將各科的學習記錄分開管理，不會混淆。

- 支援建立多個科目（如：數學、英文、歷史）
- 每個科目下可新增多篇筆記
- 支援富文字編輯（標題、粗體、條列、程式碼區塊）

---

#### 功能 2：筆記內容統整摘要

> 作為**學生**，我希望能**自動統整某科目的筆記要點**，以便在考前快速複習，不需要逐篇翻閱。

- 可針對單一科目或所有筆記產出統整摘要頁
- 摘要顯示每篇筆記的標題與關鍵條列重點
- 提供「統整視圖」與「詳細視圖」切換

---

#### 功能 3：共用連結協作編輯

> 作為**學生**，我希望**透過分享網址讓同學一起編輯筆記**，以便進行小組學習與共同整理，不需要建立帳號。

- 每份筆記（或科目）可產生唯一的分享連結
- 擁有連結者可直接進入並編輯，無需登入
- 可設定連結的權限（**可編輯** / **唯讀**）
- 可隨時撤銷或重新產生連結

---

#### 功能 4：筆記全文搜尋

> 作為**學生**，我希望**用關鍵字搜尋所有筆記的內容**，以便快速定位到想找的知識點，節省翻找時間。

- 支援跨科目全文關鍵字搜尋
- 搜尋結果高亮顯示關鍵字
- 可依科目篩選搜尋範圍

---

#### 功能 5：筆記標籤與分類管理

> 作為**學生**，我希望能**為每篇筆記加上標籤（如：重要、易錯、待複習）**，以便依標籤分類快速找到需要加強的筆記。

- 支援自訂標籤，顏色可自選
- 可依標籤篩選、瀏覽筆記
- 標籤統計頁顯示各標籤筆記數量

---

#### 功能 6：管理者後台

> 作為**管理者**，我希望能**管理所有使用者的科目與筆記**，以便維護系統內容品質，並在必要時刪除不當內容。

- 管理者可瀏覽所有科目與筆記列表
- 可刪除或停用特定筆記
- 可管理共用連結（查看、撤銷）

---

#### 功能 7：筆記匯出

> 作為**學生**，我希望能**將筆記匯出為 PDF 或 Markdown 格式**，以便離線使用或列印成紙本複習。

- 支援單篇或整個科目匯出
- 匯出 PDF 保留格式
- 匯出 Markdown 供其他工具使用

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Flask（Python） |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + Vanilla CSS + JavaScript |
| 部署環境 | 本機或學校伺服器 |

### 3.2 效能需求

- 頁面載入時間應在 **2 秒內**（一般網路環境）
- 支援至少 **50 位同時在線**使用者
- 搜尋功能回應時間應在 **1 秒內**

### 3.3 安全考量

- 共用連結使用 **UUID** 作為 Token，防止猜測
- 管理者帳號需密碼保護（Hash 儲存）
- 防止 XSS：所有使用者輸入需進行 Sanitize
- 防止 SQL Injection：使用 ORM 或參數化查詢

### 3.4 可用性需求

- 響應式設計，支援桌機與平板瀏覽
- 介面使用繁體中文
- 支援現代瀏覽器（Chrome、Edge、Firefox 最新版）
